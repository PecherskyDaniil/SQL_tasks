### Текст задания
Бизнес-задача: Начальник смены хочет видеть, какую долю от суточной добычи обеспечивает каждая единица оборудования.

Требования:

Для даты 20240115, смены 1, выведите:
Название оборудования
Тонны добычи
Общую сумму добычи за смену (через SUM() OVER())
Процент от общей добычи (округлить до 1 десятичного)
Отсортируйте по убыванию добычи
Ожидаемый результат: таблица с 4 столбцами, где сумма процентов равна 100%.
### Решение
```
select de.equipment_name,
	   sum(fp.tons_mined) as tons_of_equipment,
	   SUM(SUM(fp.tons_mined)) OVER() AS sum_tons,
	   ROUND(100 * SUM(fp.tons_mined) / SUM(SUM(fp.tons_mined)) OVER(), 1) AS percent
from fact_production fp 
join dim_equipment de on fp.equipment_id=de.equipment_id
where fp.date_id=20240115 and fp.shift_id=1
group by de.equipment_name
order by tons_of_equipment DESC;
```

## Задание 2
### Текст задания
Бизнес-задача: Построить график нарастающей добычи за январь 2024 года с разбивкой по шахтам.

Требования:

Для каждой шахты за январь 2024 рассчитайте:
Дату
Суточную добычу
Нарастающий итог добычи с начала месяца (PARTITION BY mine_id, ORDER BY full_date)
Отсортируйте по шахте и дате
Подсказка: Используйте SUM(SUM(...)) OVER (PARTITION BY ... ORDER BY ...) с предварительным GROUP BY.
### Решение
```
SELECT 
    dm.mine_name,
    dd.full_date,
    SUM(fp.tons_mined) AS daily_tons,
    SUM(SUM(fp.tons_mined)) OVER (
        PARTITION BY dm.mine_id 
        ORDER BY dd.full_date
    ) AS running_total
FROM fact_production fp
JOIN dim_mine dm ON fp.mine_id = dm.mine_id
JOIN dim_date dd ON fp.date_id = dd.date_id
WHERE dd.year = 2024 AND dd.month = 1
GROUP BY dm.mine_id, dm.mine_name, dd.full_date
ORDER BY dm.mine_name, dd.full_date;
```

## Задание 3
### Текст задания
Бизнес-задача: Логист хочет отслеживать тренд суточного расхода топлива с помощью 7-дневного скользящего среднего.

Требования:

Для шахты 1 за I квартал 2024 рассчитайте:
Дату
Суточный расход топлива (fuel_consumed_l)
7-дневное скользящее среднее (ROWS BETWEEN 6 PRECEDING AND CURRENT ROW)
14-дневное скользящее среднее
Округлите средние до 2 знаков
Вопрос: Почему первые 6 значений скользящего среднего за 7 дней рассчитаны по меньшему количеству строк? Как это влияет на точность?
### Решение
```
select dd.full_date, 
	   sum(fp.fuel_consumed_l) as sum_fuel_consumed, 
	   ROUND(AVG(SUM(fp.fuel_consumed_l)) 
	   		over(
	   			order by dd.full_date  
	   			ROWS BETWEEN 6 PRECEDING AND CURRENT row
	   			),2) as ma_7,
	   ROUND(AVG(SUM(fp.fuel_consumed_l)) 
	   		over(
	   			order by dd.full_date  
	   			ROWS BETWEEN 13 PRECEDING AND CURRENT row
	   			),2) as ma_14			
from fact_production fp 
join dim_date dd on fp.date_id=dd.date_id
where fp.mine_id=1 and dd.full_date<'2024-04-01' and dd.full_date>='2024-01-01'
group by dd.full_date
;
```

## Задание 4
### Текст задания
Бизнес-задача: HR-служба хочет ранжировать операторов по производительности внутри каждого типа оборудования для I полугодия 2024.

Требования:

Для каждого оператора рассчитайте:
ФИО оператора (фамилия + инициал)
Тип оборудования
Суммарную добычу за период
Ранг внутри типа оборудования (RANK() OVER(PARTITION BY ... ORDER BY ...))
Плотный ранг (DENSE_RANK)
Квартиль (NTILE(4))
Выведите только операторов с рангом (RANK) <= 5
Отсортируйте по типу оборудования и рангу
### Решение
```
with operator_stats as (select CONCAT(t.first_name,' ',t.last_name) as full_name,
	   det.type_name,
	   sum(fp.tons_mined) as sum_tons, 
	   RANK() OVER(PARTITION BY det.type_name ORDER BY sum(fp.tons_mined) DESC) as rank_over_type, 
	   dense_rank() over (PARTITION BY det.type_name 
            ORDER BY SUM(fp.tons_mined) DESC) as all_dense_rank,
	   NTILE(4) over (PARTITION BY det.type_name 
            ORDER BY SUM(fp.tons_mined) DESC) as qurtile
from fact_production fp 
join dim_equipment de on fp.equipment_id=de.equipment_id 
join dim_equipment_type det  on de.equipment_type_id=det.equipment_type_id 
join dim_operator t on fp.operator_id =t.operator_id 
where fp.date_id>20240101 and fp.date_id<20240701
group by det.type_name,t.first_name,t.last_name   
)
select * 
from operator_stats
where rank_over_type<=5
order by rank_over_type, type_name;
```

## Задание 5
### Текст задания
Бизнес-задача: Проанализировать разницу в добыче между дневной и ночной сменами за январь 2024, шахта 1.

Требования:

Для каждой комбинации дата+смена рассчитайте:
Суммарную добычу за смену
Добычу предыдущей смены (LAG)
Процент смены от суточной добычи (SUM OVER PARTITION BY full_date)
7-дневное скользящее среднее для каждой смены отдельно
Используйте именованные окна (WINDOW)
Отсортируйте по дате и смене
### Решение
```
select dd.full_date, 
	   ds.shift_name,
	   SUM(fp.tons_mined),
	   LAG(SUM(tons_mined)) OVER w_lag as prev_month_tons,
	   ROUND(100*SUM(fp.tons_mined)/SUM(sum(fp.tons_mined)) over w_perc,1),
	   AVG(SUM(tons_mined)) OVER w_ma
from fact_production fp 
join dim_shift ds on fp.shift_id=ds.shift_id
join dim_date dd on fp.date_id=dd.date_id
where fp.date_id>20240101 and fp.date_id<20240201 and fp.mine_id=1
group by dd.full_date, ds.shift_name 
WINDOW w_lag AS (order by dd.full_date, ds.shift_name),
	   w_perc as  (PARTITION BY dd.full_date),
	   w_ma as (partition by ds.shift_name ROWS BETWEEN 6 PRECEDING AND CURRENT row)
order by dd.full_date, ds.shift_name
;
```

Q1 = PERCENTILE_CONT(0.25)
Q3 = PERCENTILE_CONT(0.75)
IQR = Q3 - Q1
Пометьте пробы как выбросы, если:
fe_content < Q1 - 1.5 * IQR — «Выброс (низ)»
fe_content > Q3 + 1.5 * IQR — «Выброс (верх)»
Выведите только пробы-выбросы с расшифровкой (шахта, дата, номер пробы, содержание Fe, статус)

### Решение
```
WITH anomaly_stats AS (
    -- Считаем перцентили один раз для каждой шахты
    SELECT 
        mine_id,
        PERCENTILE_CONT(0.25) WITHIN GROUP (ORDER BY fe_content) AS Q1,
        PERCENTILE_CONT(0.75) WITHIN GROUP (ORDER BY fe_content) AS Q3,

)

	dd.full_date,
	foq.quality_id,
	foq.fe_content,

	a_s.Q3 + 1.5 * a_s.IQR
from fact_ore_quality foq
```

### Текст задания
Бизнес-задача: Определить 80% причин простоев по принципу Парето для приоритизации мероприятий по повышению надёжности.

Требования:

За I полугодие 2024 для каждой причины простоя рассчитайте:
Общее время простоя (часы)
Процент от общего времени простоев
Нарастающий процент (кумулятивный)
Категория Парето: «A» (до 80%), «B» (80-95%), «C» (95-100%)
Используйте SUM() OVER (ORDER BY ... DESC) для нарастающего итога
Отсортируйте по убыванию времени простоя
Ожидаемый результат: Таблица, где видно, какие 20% причин вызывают 80% простоев.
### Решение
```
WITH ReasonTotals AS (
    SELECT 
        dr.reason_name,
        SUM(fd.duration_min) / 60.0 AS total_hours,
        100.0 * SUM(fd.duration_min) / SUM(SUM(fd.duration_min)) OVER() AS pct
    FROM fact_equipment_downtime fd
    JOIN dim_downtime_reason dr ON fd.reason_id = dr.reason_id
    JOIN dim_date dd ON fd.date_id = dd.date_id
    WHERE dd.year = 2024 AND dd.month <= 6
    GROUP BY dr.reason_name
),
WithCumulative AS (
    SELECT 
        *,
        SUM(pct) OVER (ORDER BY total_hours DESC) AS cumulative_pct
    FROM ReasonTotals
)
SELECT 
    reason_name,
    ROUND(total_hours, 1) AS total_hours,
    ROUND(pct, 2) AS pct,
    ROUND(cumulative_pct, 2) AS cumulative_pct,
    CASE 
        WHEN cumulative_pct <= 80 THEN 'A'
        WHEN cumulative_pct <= 95 THEN 'B'
        ELSE 'C'
    END AS pareto_category
FROM WithCumulative
ORDER BY total_hours DESC;
```

Бизнес-задача: Для предотвращения аварий необходимо выявлять аномальные показания датчиков (температура двигателя, вибрация).

Требования:

pct_rank > 0.95 — «ОПАСНОСТЬ»
pct_rank > 0.85 — «ВНИМАНИЕ»

Используйте именованные окна для сокращения дублирования

### Решение
```
WITH SensorAnalysis AS (
    SELECT 
        ds.sensor_code,
        ft.date_id,
        ft.time_id,
        ft.sensor_value AS current_value,
        AVG(ft.sensor_value) OVER w8 AS moving_avg_8,
        STDDEV(ft.sensor_value) OVER w8 AS moving_stddev_8,
        ft.sensor_value - LAG(ft.sensor_value) OVER w_seq AS delta,
        PERCENT_RANK() OVER (
            PARTITION BY ft.sensor_id 
            ORDER BY ft.sensor_value
        ) AS pct_rank
    FROM fact_equipment_telemetry ft
    JOIN dim_sensor ds ON ft.sensor_id = ds.sensor_id
    WHERE ft.equipment_id = 1 
      AND ft.date_id BETWEEN 20240101 AND 20240107
    WINDOW 
        w8 AS (PARTITION BY ft.sensor_id 
               ORDER BY ft.date_id, ft.time_id 
               ROWS BETWEEN 7 PRECEDING AND CURRENT ROW),
        w_seq AS (PARTITION BY ft.sensor_id 
                  ORDER BY ft.date_id, ft.time_id)
)
SELECT 
    *,
    CASE 
        WHEN pct_rank > 0.95 THEN 'ОПАСНОСТЬ'
        WHEN pct_rank > 0.85 THEN 'ВНИМАНИЕ'
        ELSE 'Норма'
```

## Задание 12
### Текст задания
Бизнес-задача: Подготовить данные для ежедневного дашборда директора шахты, объединяющего все ключевые метрики с трендами.

Требования:

Для шахты 1 за январь 2024 постройте одним запросом:
Дату
Суточную добычу
Добычу предыдущего дня (LAG)
Изменение день-к-дню (%)
7-дневное скользящее среднее добычи
Нарастающий итог добычи с начала месяца
Ранг дня по добыче за месяц (RANK)
NTILE(3) для категоризации: «Высокая», «Средняя», «Низкая» добыча
Медианное значение добычи за весь месяц (через оконную PERCENTILE_CONT)
Отклонение от медианы (%)
Используйте минимум 2 именованных окна
Добавьте текстовый столбец «trend»: «рост» / «снижение» / «стабильно» (если изменение < 5%)
### Решение
```
WITH daily AS (
    SELECT 
        d.full_date, 
        SUM(fp.tons_mined) AS tons
    FROM fact_production fp
    JOIN dim_date d ON d.date_id = fp.date_id
    WHERE fp.mine_id = 1 AND d.year = 2024 AND d.month = 1
    GROUP BY d.full_date
),
stats AS (
    SELECT
        full_date,
        tons,
        LAG(tons) OVER w_seq AS prev_day,
        AVG(tons) OVER w7 AS moving_avg_7d,
        SUM(tons) OVER w_cum AS running_total,
        RANK() OVER (ORDER BY tons DESC) AS day_rank,
        NTILE(3) OVER (ORDER BY tons DESC) AS production_tier,
        (SELECT PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY tons) FROM daily) AS month_median
    FROM daily
    WINDOW 
        w_seq AS (ORDER BY full_date),
        w7    AS (ORDER BY full_date ROWS BETWEEN 6 PRECEDING AND CURRENT ROW),
        w_cum AS (ORDER BY full_date ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW)
)
SELECT
    full_date,
    tons,
    prev_day,
    ROUND(100.0 * (tons - prev_day) / NULLIF(prev_day, 0), 1) AS daily_change_pct,
    ROUND(moving_avg_7d, 1) AS avg_7d,
    running_total,
    CASE production_tier 
        WHEN 1 THEN 'Высокая' WHEN 2 THEN 'Средняя' ELSE 'Низкая' 
    END AS tier_name,
    ROUND((100 * (tons - month_median) / month_median)::numeric, 1) AS vs_median_pct,
    CASE 
        WHEN (100.0 * (tons - prev_day) / NULLIF(prev_day, 0)) > 5 THEN 'Рост'
        WHEN (100.0 * (tons - prev_day) / NULLIF(prev_day, 0)) < -5 THEN 'Снижение'
        ELSE 'Стабильно'
    END AS trend
FROM stats
ORDER BY full_date;
```
